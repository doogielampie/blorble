# Blorble v2.1 Implementation Plan — UI refresh (landing, header, dialogs)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Raster checkpoints are orchestrator-LOOKed. The user-approved mockups (chat frames A–E, 2026-07-02) are the binding visual spec; this plan encodes their structure.

**Goal:** Ship the v2.1 UI refresh: a level-select landing screen (replacing the gate screens and in-game tabs), a one-row in-game header with date/practice next to the timer, a hint pill under the board, a trimmed how-to dialog (small ×, never auto-opens), and a minimal-but-silly results dialog that IS the share card (mascot + quip + big time; streak/best removed from display). **One layout everywhere** — desktop shows the same centered column as mobile (user decision 2026-07-02, superseding the earlier side-rail choice).

**Version call:** v2.1, not v3 — presentation and copy only. No game mechanics, no board/seed changes, no save-schema changes (streak/best stay *recorded* in storage, just not displayed; `seenHowTo` becomes vestigial but stays in the schema).

## Context

User playtested v2 and gave a design-refinement list; mockups were built and approved interactively with the decisions locked: (1) **landing screen** over in-game tabs (frame A) — per-level cards with done-states and practice links; (2) **hint pill under the board** (frame B); (3) **quips** on the results screen (frame E) — performance-aware one-liners; (4) **identical layout on desktop and mobile** (the side-rail mockup was considered and REJECTED at plan approval — do not build frame C). Also locked by the user's list: remove the how-to's win-condition/hint/clock paragraphs and Got-it button (small × instead); remove both gate taglines; date or `practice` as plain text right of the timer (no blue pill badge); never auto-open the how-to; results shows only the current session (no streak/best), no duplicated info, × instead of Close.

## Global Constraints (v2 constraints carry over; deltas below)

- All v2 invariants hold: seeds/boards untouched, storage key + schema `v:2` untouched, `recordWin`/streak/best still recorded, share TEXT format unchanged (`Label · Mon D · ⏱ m:ss · ✨|💡n ✖️n`), dev-param isolation (`date/solve/autoplay/hint` → `dev.*`), midnight rollover reload, mobile one-screen at 390×844 AND 375×667, `pnpm check` green every commit, TDD for pure code, phone rasters via `tools/phone-shot.sh` (also used for the desktop checkpoint at 1024×768), commit trailer `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- "Clock starts when you peek" survives: the landing card's Play/Resume button IS the gate — `reveal()` (with its daily-record creation + rollover guard) runs on entering the board, exactly as before.
- Cross-tab wall clocks still don't pause; with taglines gone this is now surfaced ONLY in the results/README copy — acceptable (user removed the copy knowingly).
- The `?autoplay=1` dev param now means "skip the landing, reveal the mode's board" (needed by every raster); `?practice=1` skips landing into practice. Plain URL always shows the landing.
- No auto-open of the how-to anywhere. `seenHowTo` stops being written (field stays for schema stability).
- Icons: inline SVG paths (no icon-font dependency): × (close), ‹ (back), bulb (hint). Keep them tiny helpers in main.ts.

```ts
// card.ts — NEW: performance-aware quip, deterministic per (dateIso, mode, outcome)
export const quip = (info: { isoDate: string; label: string; hints: number; wrongs: number }): string => {
  const CLEAN = ["clean blorbing!", "flawless. the Blorbs bow.", "not a single grump."];
  const HINTY = ["hints were consumed.", "a little help never hurt.", "the bulb did its part."];
  const WRONGY = ["a scenic route!", "the Blorbs forgive you.", "grumps happened."];
  const pool = info.hints === 0 && info.wrongs === 0 ? CLEAN : info.hints > 0 ? HINTY : WRONGY;
  let h = 0;
  const key = `${info.isoDate}:${info.label}`;
  for (let i = 0; i < key.length; i++) h = (h * 31 + key.charCodeAt(i)) >>> 0;
  return pool[h % pool.length]!;
};
```
