## Toolchain gotchas (paid for — don't rediscover)

- Phone rasters MUST use `tools/phone-shot.sh OUT.png W H "/blorble/?query" [PROFILE] [PORT]` (headless
  Chrome clamps width ≥500px; PNG is (W+200)×(H+200), judge top-left W×H). **Pin [PORT] whenever two shots
  share a [PROFILE]** — the default port is random and localStorage is origin-scoped incl. the port.
- The how-to shot uses `?howto=1`; the first-visit nudge shot is a fresh-profile boot (e.g.
  `?date=2026-07-04`, no `autoplay`). Never screenshot the Vite dev server.
- Static-file rasters (e.g. the mockup): headless Chrome tends to hang on exit — run it backgrounded, poll
  for the PNG, then `pkill -f 'headless=new'`.
- Global `pnpm` (no corepack). Vitest on Node 25 sometimes SIGABRTs after green output — rerun once.
- NO `gh` CLI. Deploy = push to `main`; verify via live-bundle polling (grep for a v2.4 marker such as
  `nobody's the odd one out` or `PODS FOUND`). GitHub Pages `deploy-pages` has flaked transiently before
  ("Deployment failed, try again later") — if it does, re-trigger with an empty commit; the current good
  bundle keeps serving in the meantime.
