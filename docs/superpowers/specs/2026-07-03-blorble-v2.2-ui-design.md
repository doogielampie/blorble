# Blorble v2.2 UI design — board-first + curtain, results polish

Approved in brainstorming 2026-07-03 (post art-v2 ship). Mockup rounds with user LOOK gates
decide all exact visuals; this spec fixes structure and behaviour. Implementation planning
(writing-plans) happens only after the user LOCKS the mockups.

## 1. Board-first single screen + top bar

- The game screen is the only screen; the v2.1 landing screen is removed.
- Boot lands on the board: last-played mode if stored, else Blorblet. Last-mode is an
  ADDITIVE optional field in the v:2 saved state — storage key `blorble.v1` and all existing
  fields stay byte-compatible; old saves must load unchanged.
- Top bar (mockups decide final look): "Blorble" wordmark left; segmented mode toggle
  `Blorblet | Blorble` with ✓ on completed modes; right side date (`Jul 3` / `practice`) ·
  timer · ? chip. The back button dies. ? opens the existing how-to dialog (this delivers
  "how-to reachable in-game").
- Toggle switches modes any time; per-mode board/timer state already independent; clocks
  don't pause (existing decision, unchanged).

## 2. Curtain (timer fairness — the reason the landing existed)

- A fresh daily board renders face-down/obscured (card-back treatment decided in mockups)
  under a compact overlay: mini mascot + "9 Blorbs · 4 Sets" (per mode) + big Play + small
  "practice instead" link. First tap lifts the curtain and starts the clock.
- In-progress board: no curtain (resume instantly). Completed board: no curtain, revealed
  solved, existing `n/n · time · Results` row.
- Dev/raster params keep semantics: `?autoplay=1` skips the curtain (replaces "skips the
  landing"); `?practice=1`, `?mode=`, `?date=`, `?solve=`, `?hint=` unchanged.

## 3. Results dialog polish

- Actions: primary **Copy image** (on clipboard-image failure, auto-fallback to PNG
  download — no visible save button); small underlined **copy text** link. "save image
  instead" removed.
- Look (mockups decide exact values): consistent vertical rhythm; bubble tail aligned to the
  mascot; softer close affordance; marks line restyled so it doesn't read as a link; dialog
  composition mirrors the share-card PNG (what you see ≈ what you copy).

## 4. Small fixes riding along

- How-to legend: widen the label column (`.legend` 64px → fits "Antennae" with clear gap);
  verify at 330px dialog width.
- Optional rider (decide during mockups): suppress the focus ring on `.dialog-scroll`.

## 5. Unchanged (hard constraints)

Game logic, seeds/boards, storage key + existing schema fields, share text format, the art
(`renderBlorb` untouched), one-layout-everywhere (desktop side rail stays rejected).

## Process

Mockup page in `design/blorble-v2.2-ui/` → raster at 390×844 + 375×667 → user judges rounds
in chat → LOCKED → writing-plans → subagent-driven execution → both-phone raster DoD →
deploy + live smoke (including an existing-save resume check).
