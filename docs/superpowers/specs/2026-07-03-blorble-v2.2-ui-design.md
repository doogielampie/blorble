# Blorble v2.2 UI design — board-first + curtain, results polish

Approved in brainstorming 2026-07-03 (post art-v2 ship). **DESIGN LOCKED 2026-07-03** after 5
mockup rounds — the locked reference is `design/blorble-v2.2-ui/mockups.html` (rows: R2 lock
sheet L1-L5 = `r2.png`; practice flow P1 + P2c + P3 = `r4.png`). Where this spec and the locked
mockups disagree, the mockups govern. Execution runs in a FRESH session:
`docs/superpowers/V2.2-UI-HANDOFF.md` is the entry point (writing-plans → subagent-driven
execution → both-phone raster DoD → deploy).

## 1. Board-first single screen + top bar

- The game screen is the only screen; the v2.1 landing screen is removed.
- Boot lands on the board: last-played mode if stored, else Blorblet. Last-mode is an
  ADDITIVE optional field in the v:2 saved state — storage key `blorble.v1` and all existing
  fields stay byte-compatible; old saves must load unchanged.
- Top bar LOCKED (mockup F1 + G1): row 1 = "Blorble" wordmark left · **timer true-centered**
  (grid 1fr/auto/1fr; EMPTY until Play starts the clock; final time persists when done) ·
  date + ? chip right. Row 2 = full-width segmented toggle `Blorblet | Blorble` with green ✓
  on completed modes. The back button dies. ? opens the existing how-to dialog.
- Toggle ALWAYS navigates to daily boards (never practice); per-mode board/timer state already
  independent; clocks don't pause (existing decision, unchanged).
- First visit (no saves): boots to the Blorblet fresh-daily curtain.
- Date rollover: when the UTC day rolls (existing `dateRolled` logic), completion ✓s reset and
  fresh curtains return for the new boards — the curtain absorbs v2.1's "New board" chip role.

## 2. Curtain (timer fairness — the reason the landing existed)

- A fresh daily board renders face-down — LOCKED back treatment: pale body-silhouette
  (`#e7dfd3` fill, no face) on the tile — under a compact overlay card (paper bg, 2.5px ink
  border): mini mascot + mode name + "9 Blorbs · 4 Sets" + big Play + small "practice
  instead" link. First tap lifts the curtain and starts the clock.
- In-progress board: no curtain (resume instantly). Completed board: no curtain, revealed
  solved, results row LOCKED as `n/n ✓ · [Results] · practice` — the duplicate time is
  dropped (final time lives in the top bar), and `practice` is a quiet underlined link.
- Dev/raster params keep semantics: `?autoplay=1` skips the curtain (replaces "skips the
  landing"); `?practice=1`, `?mode=`, `?date=`, `?solve=`, `?hint=` unchanged.

## 3. Practice flow (LOCKED: P1 + P2c + P3)

- Practice is a quiet detour; the daily is the spotlight. Entries: the curtain's "practice
  instead" link (pre-daily) and the done-row's `practice` link (post-daily). Deliberately NO
  practice entry mid-daily.
- In practice (P2c): the toggle grows a temporary third segment `Practice`, lit as active;
  the two daily segments un-highlight. Exit = tap either daily segment (returns to that
  mode's daily in whatever state: curtain / mid-board / done). The segment exists only while
  practicing. Date slot keeps showing the date; timer runs centered as usual.
- Practice results dialog (P3): primary **New board** (deals a fresh practice board);
  context line `Practice · <mode>`; single demoted `copy image` link (no copy-text, no
  save). Practice boards start instantly — no curtain (re-rolls stay frictionless; practice
  times aren't the social artifact).

## 4. Daily results dialog polish (LOCKED: F6)

- Actions: primary **Copy image** (on clipboard-image failure, auto-fallback to PNG
  download — no visible save button); small underlined **copy text** link. "save image
  instead" removed. (The old "Play again" button's job moves to the done-row practice link.)
- Look (F6 "tightened"): current mascot+bubble structure with consistent vertical rhythm;
  soft borderless × ; marks line keeps accent only on the emphasis ("no misses"); dialog
  stays share-only — navigation lives in the row beneath it.

## 5. Small fixes riding along

- How-to legend: LOCKED at 82px label column + 8px column gap; verify at 330px dialog width.
- The stacked bar costs ~40px vs the old header → board cap `46dvh` → **~43dvh** for the
  one-screen fit at 375×667 (re-verify with phone-shot; tune ±1dvh if needed).
- Optional rider: suppress the focus ring on `.dialog-scroll`.

## 6. Unchanged (hard constraints)

Game logic, seeds/boards, storage key + existing schema fields, share text format, the art
(`renderBlorb` untouched), one-layout-everywhere (desktop side rail stays rejected).
